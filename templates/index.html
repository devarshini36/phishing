Project Overview

This system is designed to detect phishing websites by analyzing various URL-based features using a machine learning model. The model is trained to differentiate between legitimate and phishing URLs.

How It Works

  1. Feature Extraction: Extracts URL-based attributes such as domain length, number of dots, query parameters, etc.
  2. Feature Selection: Uses Bat Algorithm to eliminate irrelevant features, improving efficiency.
  3. Model Training: Utilizes LightGBM classifier, trained with labeled data to enhance accuracy.
  4. Prediction: Classifies input URLs as phishing or legitimate in real time.

Machine Learning Model

The model is trained on a dataset containing multiple features extracted from URLs. It uses LightGBM as the primary classifier, offering high accuracy and low computational cost.

  • Feature Engineering: Extracts attributes from the URL to generate meaningful input features.
  • Data Normalization: Min-Max Scaling is applied to standardize values.
  • Feature Selection: Uses the Bat Algorithm to select the most important attributes.
  • Classification: The LightGBM model makes final predictions based on extracted and selected features.

Dataset Details

The dataset used for training consists of URLs labeled as either phishing or legitimate. Features considered include:

  • URL Length: Longer URLs tend to be more suspicious.
  • Number of Dots: More dots indicate higher chances of phishing.
  • Domain Entropy: Higher entropy values suggest randomness in the domain name, often linked to phishing attempts.
  • Special Characters: Presence of symbols like '@', '-', '?' often signifies phishing tactics.

Experimental Results

The model was tested on real-world datasets, achieving high detection accuracy. Key performance metrics include:

  • Accuracy: 98.7%
  • Precision: 97.8%
  • Recall: 98.2%
  • F1 Score: 98.0%

Future Enhancements

Potential improvements to the phishing detection system include:

  • Integration of Natural Language Processing (NLP) to analyze website content.
  • Enhancing real-time phishing detection by incorporating WHOIS data.
  • Deploying deep learning models like CNNs and Transformers for improved classification.
